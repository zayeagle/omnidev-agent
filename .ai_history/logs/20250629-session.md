# 2025-06-29 开发协作日志

## Turn 1

**用户：** 目前 make deploy 的时候二进制文件会存放在哪里

**助手：** 说明 `make deploy` = rebuild + install-binary + config。二进制先编译到 `bin/omnidev-agent`，再安装到 `$HOME/.local/bin/omnidev-agent`（可用 `PREFIX` 覆盖）。

---

## Turn 2

**用户：** 配置文件可以和二进制文件放在一起吗

**助手：** 当前不行。配置按 cwd 读 `./.omnidev-agent.json`，全局在 `~/.omnidev-agent/config.json`；`make deploy` 的 config 写在项目根，不与二进制同目录。

---

## Turn 3

**用户：** 安装一次、到处可用，想要的是这个效果

**决策：** 将 `make config` / `make deploy` 改为初始化全局配置 `~/.omnidev-agent/config.json`，实现任意目录运行 `omnidev-agent` 即可加载 API 等设置。

**改动：**
- `Makefile`：`config` → 全局配置（chmod 600）；新增 `config-local` 保留项目级可选覆盖；`uninstall` 修正路径提示
- `scripts/install.ps1`：Windows 安装时同步创建全局配置
- `README.md`：Deployment 章节更新

**使用：**
```bash
make deploy
# 编辑 ~/.omnidev-agent/config.json
omnidev-agent   # 任意目录可用
```

## Turn 4

**用户：** 构建部署的时候全局和项目目录都会创建配置文件吗？

**助手：** 不会同时创建。`make deploy` 只初始化全局 `~/.omnidev-agent/config.json`；项目级 `./.omnidev-agent.json` 需单独 `make config-local`。

---

## Turn 5

**用户：** 为什么不能全局和项目配置都创建？

**助手：** 此前是刻意分离，非技术限制。已改 `deploy: rebuild install-binary config config-local`，两者都会在缺失时从 sample 创建。
